# Data — Database Context & Configuration

This folder contains the Entity Framework Core `ApplicationDbContext` which defines the database schema, relationships, constraints, and seed data.

## ApplicationDbContext.cs

Extends `IdentityDbContext<ApplicationUser>` (from ASP.NET Core Identity), adding the studio-specific entities.

### DbSets (Tables)

| DbSet | Model | Table | Description |
|-------|-------|-------|-------------|
| ClassTypes | ClassType | ClassTypes | Art class categories (Painting, Yoga, etc.) |
| Sessions | Session | Sessions | Scheduled class sessions |
| Students | Student | Students | Student profiles |
| Attendances | Attendance | Attendances | Student attendance per session |
| StudentLinkRequests | StudentLinkRequest | StudentLinkRequests | Account linking requests |
| PackageDefinitions | PackageDefinition | PackageDefinitions | Package templates |
| StudentPackages | StudentPackage | StudentPackages | Package enrollments |
| Payments | Payment | Payments | Payment records |

Plus the Identity tables: AspNetUsers, AspNetRoles, AspNetUserRoles, etc.

### Relationship Configuration (Fluent API)

All relationships are configured in `OnModelCreating()` using Fluent API rather than data annotations for clarity.

**Session relationships:**
```
Session → ClassType       (Required, Restrict delete)
Session → CreatedBy       (Required, Restrict delete)
Session.StartDateTime     (Has index for date range queries)
```

**Student relationships:**
```
Student → User            (Optional 1:1, SetNull on delete)
Student → Parent          (Optional M:1, SetNull on delete)
Student.Email             (Unique index)
```

**Attendance relationships:**
```
Attendance → Session      (Required, Cascade delete)
Attendance → Student      (Required, Cascade delete)
(SessionId, StudentId)    (Unique constraint — one record per student per session)
```

**StudentLinkRequest relationships:**
```
StudentLinkRequest → RequestedBy   (Required, Cascade delete)
StudentLinkRequest → Student       (Optional, SetNull on delete)
StudentLinkRequest → ReviewedBy    (Optional, SetNull on delete)
```

**Billing relationships:**
```
PackageDefinition → ClassType      (Required, Restrict delete)
StudentPackage → Student           (Required, Cascade delete)
StudentPackage → PackageDefinition (Required, Restrict delete)
StudentPackage → CreatedBy         (Required, Restrict delete)
(StudentId, PackageDefinitionId, BillingYear, BillingMonth) (Unique constraint)

Payment → Student                  (Required, Cascade delete)
Payment → RecordedBy               (Required, Restrict delete)
```

### Decimal Column Configuration

SQLite does not have a native decimal type. All decimal properties are explicitly configured:
```csharp
.HasColumnType("decimal(10,2)")
```

Applies to: `PackageDefinition.Price`, `StudentPackage.PackagePrice`, `Payment.Amount`, `Payment.Discount`, `ClassType.DefaultSessionPrice`

### Seed Data

On database creation, the following data is seeded:

**Roles:**
- Administrator
- Instructor
- Student
- Parent

**Default ClassType:**
- Name: "Painting"
- Color: "#4CAF50"
- TargetAgeGroup: Kids
- DefaultSessionPrice: 0 (should be configured by admin)

### Delete Behavior Summary

| Relationship | On Delete | Reason |
|-------------|-----------|--------|
| Session → Attendances | Cascade | No orphan attendance records |
| Student → Attendances | Cascade | No orphan attendance records |
| Student → Payments | Cascade | Clean up financial records |
| Student → StudentPackages | Cascade | Clean up enrollments |
| StudentLinkRequest → Student | SetNull | Preserve request history |
| StudentLinkRequest → ReviewedBy | SetNull | Preserve request if admin deleted |
| Student → User | SetNull | Student profile survives user deletion |
| Student → Parent | SetNull | Student profile survives parent deletion |
| Session → ClassType | Restrict | Cannot delete class type with sessions |
| PackageDefinition → ClassType | Restrict | Cannot delete class type with packages |

### Running Migrations

```bash
# Apply all pending migrations
dotnet ef database update

# Create a new migration after model changes
dotnet ef migrations add MigrationName

# Remove the last migration (if not applied)
dotnet ef migrations remove

# Generate SQL script (for production deployment)
dotnet ef migrations script
```
